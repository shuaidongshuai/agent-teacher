## 1. 这个目录解决什么问题

`99-内容维护` 不是课程主线，而是仓库的维护规范区。

它解决的是：

- 后续新增文档怎么写
- 什么文档需要编号
- 怎样做最小质检

## 2. 适用人群

适合维护这套课程的人，尤其适合后续继续新增教学文档、项目 README 或配套讲义的人。

## 4. 什么时候需要编号

下面两类文档需要编号：

1. 同一主题下存在明确先后顺序的教学讲义
2. 同一组文档需要在 README 里声明阅读顺序

下面两类内容默认不编号：

1. 项目代码目录
2. 资料收集模板、术语表、周计划模板、运行产物

## 5. 新增文档的默认流程

建议按下面流程维护：

1. 外部资料先进入 [08-资料收集](/Users/chenmingdong01/Documents/AI/agent/08-资料收集/README.md)
2. 消化后再转成正式章节文档
3. 如果属于线性教学内容，就在所在目录按顺序编号
4. 更新该目录 README 的阅读顺序
5. 重要术语同步补进 [09-术语表](/Users/chenmingdong01/Documents/AI/agent/09-术语表/README.md)

## 6. 总结与下一步

后续新增内容前，先过一遍模板和质检清单，能大幅减少结构失衡和命名混乱。
