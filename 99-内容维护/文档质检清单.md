# 文档质检清单

## 目录

1. [章节级检查](#1-章节级检查)
2. [单篇文档检查](#2-单篇文档检查)
3. [项目 README 检查](#3-项目-readme-检查)
4. [命名与编号检查](#4-命名与编号检查)
5. [总结与使用建议](#5-总结与使用建议)

## 1. 章节级检查

- 是否存在 `README.md`
- README 是否声明本章定位
- README 是否包含适用人群或学习目标
- README 是否给出明确阅读顺序
- README 是否说明学完后的产出
- README 是否有总结与下一步

## 2. 单篇文档检查

- 是否有文档主标题
- 如果文档较长，是否有目录
- 是否写明学习目标或适用人群
- 是否不是只罗列定义，而是有教学展开
- 是否包含总结与下一步建议

## 3. 项目 README 检查

- 是否先讲教学价值，再讲运行方式
- 是否有前置知识
- 是否有推荐观察点
- 是否有常见失败原因
- 是否有练习任务

## 4. 命名与编号检查

- 同组文档如果有先后顺序，文件名是否带编号
- 文件名编号与 H1 标题编号是否一致
- README 中的阅读顺序是否与文件编号一致
- 是否误把项目代码目录或运行产物也做了编号

## 5. 总结与使用建议

新增或重写文档前，先对照模板；提交前，再过一遍这份清单。  
如果一篇文档不能通过这些最小检查，通常说明它还没有真正进入“可授课”状态。
