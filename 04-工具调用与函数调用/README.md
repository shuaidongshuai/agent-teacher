## 1. 本章定位

`04-工具调用与函数调用` 解决的是：

**怎样让模型安全、稳定地连接外部能力，而不是只会基于已有上下文猜答案。**

这是从“会聊天”走向“能做事”的关键一章，也是 Agent 的重要基础。

## 2. 适用人群

适合：

- 已经会写 Prompt，但应用还停留在纯文本回答的人
- 想把模型接入天气、搜索、数据库、业务 API 的开发者

## 3. 学习目标

学完这一章后，你应该能够：

1. 区分直接回答和调用工具后回答的差异
2. 设计结构化、可校验、可扩展的工具参数
3. 处理超时、空结果、格式异常等失败场景
4. 让模型基于工具结果做二次推理
5. 理解多工具场景下的选择策略和 MCP 的定位

## 5. 本章核心问题

建议围绕下面几个问题学习：

1. 模型什么时候该调用工具，什么时候不该调
2. 参数怎么设计才稳定
3. 工具返回结果后，模型怎么继续推理
4. 失败、重试、降级、幂等怎么设计
5. MCP 为什么比一次性函数调用更适合能力复用

## 7. 常见误区

- 把工具调用当成黑箱自动能力
- 工具返回后直接把原始 JSON 暴露给用户
- 失败就报错，不做重试和降级
- 见工具就调，不考虑成本和延迟
