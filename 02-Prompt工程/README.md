## 1. 本章定位

`02-Prompt工程` 解决的是：

**面对一个真实任务，怎样把目标、上下文和输出要求组织成模型更容易正确执行的输入。**

这一章不把 Prompt 当成玄学技巧，而是把它拆成任务表达、约束设计、示例设计、结构化输出和调试。

## 2. 适用人群

适合：

- 已经知道 LLM 基本原理，准备进入真实任务设计的人
- 经常觉得“模型有时好有时差”，但还不会系统调试 Prompt 的人

## 3. 学习目标

学完这一章后，你应该能够：

1. 理解 Prompt 工程是任务表达，不是咒语学
2. 把模糊需求改写成可执行任务
3. 用角色、目标、约束、输出格式降低跑偏
4. 用 few-shot 示例提升一致性
5. 用调试记录定位失败原因并持续迭代

## 5. 本章核心问题

建议围绕下面 4 个问题学习：

1. 一个任务怎样写得更清楚
2. 约束和输出格式怎样减少歧义
3. few-shot 什么时候有效，什么时候会误导模型
4. 效果差时，怎样判断是 Prompt 问题还是别的层出了问题

## 7. 常见误区

- 把 Prompt 工程理解成套模板
- 误以为 Prompt 越长越好
- 只靠反复试，不做失败定位
- 把 Prompt 问题和模型能力问题混在一起
