## 1. 适用人群

这个目录适合在学习过程中持续收集文章、课程、视频、论文和仓库的人。

## 2. 学习目标

学会把“看到资料”变成“沉淀知识”的第一步，而不是只收藏链接。

## 3. 这个目录解决什么问题

很多学习资料库最后会变成一个问题：

- 收藏很多
- 真正消化很少
- 资料和自己的笔记脱节

这个目录的作用，就是把外部资料先集中落地，再决定：

- 它值不值得细读
- 它应该归到哪个主题
- 它最后要不要转成正式教学文档

## 4. 建议使用顺序

建议按下面顺序使用：

1. 先用 [资料整理模板.md](/Users/chenmingdong01/Documents/AI/agent/08-资料收集/资料整理模板.md) 记录资料
2. 给资料标注主题、阶段和是否值得二刷
3. 真正消化后，把内容转写到对应主题目录
4. 把关键概念同步补进 `09-术语表`

## 5. 资料筛选标准

建议优先保留下面几类资料：

- 能解释底层原理的
- 能讲清工程取舍的
- 有真实失败案例和优化经验的
- 能帮助你形成项目思路的

建议少积累下面这类资料：

- 只展示效果，不解释过程
- 只讲框架名字，不讲设计原因
- 只追热点，没有长期复用价值

## 6. 总结与下一步建议

资料收集的目标不是“越多越好”，而是为后续沉淀服务。

建议你每收 5 篇资料，就至少做 1 次整理：

1. 删掉低价值链接
2. 转写 1 篇高价值理解
3. 补 3 个术语到 `09-术语表`
