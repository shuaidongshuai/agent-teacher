# LLM 基础

## 适用人群

这部分内容适合刚进入大模型学习路径的读者，尤其是：

- 想先建立整体认知，再进入 Prompt、RAG、Agent 的人

学完这一部分后，你应该能够：

1. 用自己的话说明 LLM 的基本工作机制
2. 理解 Transformer 为什么适合做语言建模

4. 区分预训练、微调、对齐分别在解决什么问题
5. 理解温度、Top-k、Top-p 等推理参数会怎样影响结果

如果把后面的 Prompt、RAG、Agent 看成“怎么把模型用起来”，那 `01-LLM基础` 讲的就是：

**模型到底是什么，它为什么能工作，它又为什么经常不稳定。**

很多初学者一上来就学框架、学 API、学案例，短期会有成就感，但一旦遇到下面这些问题就容易卡住：

- 为什么同一个 Prompt 改一点点，结果差很多？
- 为什么模型看起来懂了，但还是会答错？
- 为什么有时上下文一长，效果就明显变差？
- 为什么有些参数一调，回答就更发散或者更保守？

这些问题的底层，都和 LLM 基础认知有关。

## 4. 这一章最重要的认知主线

学 LLM 基础时，建议抓住下面 4 条主线：

### 4.1 LLM 本质上是概率模型

它不是“真的理解世界”的人类大脑，而是在已有上下文基础上，预测下一个最可能出现的 token。

### 4.2 能力来自规模、数据、结构和训练目标的共同作用

不是因为某一个技巧神奇，而是因为：

- 看了大量文本
- 用合适结构建模长距离依赖
- 在海量预测任务中形成语言规律
- 再经过后续微调与对齐，变得更像可用助手

### 4.3 它很强，但不是无限可靠

模型擅长模式拟合、语言生成、压缩表达，但不天然保证：

- 事实正确
- 逻辑严谨
- 计算准确
- 始终服从指令

### 4.4 工程效果经常受“上下文供给”限制

很多人以为模型能力只由参数规模决定，但真实应用里，效果往往还取决于：

- 给了什么上下文
- 上下文有多长
- 是否被截断
- 检索到的材料是否相关
- 输出是否被合理约束

## 5. 常见误区

### 5.1 误区一：LLM 就是“会搜索的数据库”

不是。模型不是把答案原样存起来再查出来，而是基于概率分布生成输出。

### 5.2 误区二：参数越大，就一定什么都更好

不一定。模型效果还受训练数据、对齐策略、上下文质量、调用方式影响。

### 5.3 误区三：会聊天就等于会推理

语言流畅和推理可靠是两回事。很多时候，模型只是把“看起来像正确答案”的形式生成得很好。

### 5.4 误区四：温度调低，就一定更正确

温度低通常会让输出更稳定，但“更稳定”不等于“更真实”或“更准确”。

如果你现在还在入门阶段，不要急着抠太深的数学推导。更重要的是先能回答下面几个问题：

- LLM 为什么可以看起来像在理解？

- Token 和上下文窗口为什么会直接影响产品效果和成本？
- 预训练、微调、对齐分别改变了模型什么能力？
