## 1. 本章定位

`01-LLM基础` 解决的是整套课程里最底层的问题：

**模型到底是什么，它为什么能工作，它又为什么经常不稳定。**

后面的 Prompt、RAG、工具调用、Agent，都是在“使用模型”；这一章讲的是“理解模型”。

## 2. 适用人群

适合下面两类学习者：

- 刚进入大模型方向，需要先建立整体认知的人
- 会调用模型 API，但解释不清楚模型行为原因的人

## 3. 学习目标

学完这一章后，你应该能够：

1. 用自己的话解释 LLM 的基本工作机制
2. 理解 Transformer 为什么适合做语言建模
3. 说明 Token 和上下文窗口为什么会影响效果与成本
4. 区分预训练、微调、对齐分别解决什么问题
5. 理解温度、Top-k、Top-p 等采样参数的作用

## 5. 本章核心问题

读这一章时，建议始终围绕下面 4 个问题：

1. LLM 本质上在做什么
2. 为什么模型会“看起来像理解”
3. 为什么上下文、采样和训练阶段会显著影响结果
4. 为什么“输出流畅”不等于“事实准确、逻辑可靠”

## 6. 学完后的产出

建议至少完成下面 3 个产出：

1. 一份自己的 LLM 原理解释稿
2. 一张 Token、上下文窗口和成本的关系图
3. 一份采样参数实验记录

## 7. 常见误区

- 误以为参数大就一定解决所有问题
- 误以为模型流畅就说明它理解正确
- 误以为上下文窗口只是“长度问题”，和效果关系不大
- 误以为采样参数只是风格设置，不影响稳定性
