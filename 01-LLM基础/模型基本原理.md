# 模型基本原理

## 学习目标

这篇文档帮助你先建立一个最核心的认识：

**大语言模型本质上是在上下文条件下，持续预测下一个 token。**

学完后你应该能够：

1. 解释什么叫“下一个 token 预测”
2. 理解为什么简单目标能涌现复杂能力
3. 说清楚模型的输入、输出和概率本质
4. 对 LLM 的能力边界形成基本判断

## 1. 什么是大语言模型

大语言模型，英文是 `Large Language Model`，通常缩写成 `LLM`。

“大”主要指两件事：

- 参数规模大
- 训练数据规模大

“语言模型”指的是：

**它在做关于语言序列的概率建模。**

更具体地说，当模型看到前面的内容后，它会判断：

“接下来最有可能出现的 token 是什么？”

例如输入：

`今天天气很`

模型可能会认为后面出现“好”“热”“冷”的概率比较高，而“数据库”“火箭”这样的概率比较低。

## 2. LLM 解决了什么问题

如果把传统程序理解成“人把规则写死，机器按规则执行”，那 LLM 更像是：

**机器从大量文本中自己学习规律，然后在新上下文里生成合理延续。**

它擅长的任务往往包括：

- 问答
- 总结
- 改写
- 翻译
- 分类
- 信息抽取
- 代码生成

这些任务表面不同，但底层都可以被统一成：

**根据上下文，生成最合适的后续文本。**

---

## 3. LLM 是怎么工作的

可以把整个过程理解成一个循环：

1. 用户输入一段文本
2. 文本被切成 token
3. 模型根据已有上下文计算下一个 token 的概率分布
4. 根据采样策略选出一个 token
5. 把这个 token 接到上下文后面
6. 重复这个过程，直到结束

这意味着，模型不是“一次性想完整个答案”，而是：

**一步一步往后写。**

这个认知很重要，因为它解释了很多现象：

- 为什么前面一句 Prompt 改一下，后面整段都可能变
- 为什么模型容易被前文带偏
- 为什么长输出中后半段可能逐渐跑题

## 4. 为什么“预测下一个 token”会这么强

很多人第一次听到这个原理时会觉得奇怪：

“只是猜下一个词，为什么能回答问题、写代码、做推理？”

关键在于，语言并不是随机词堆。语言背后压缩了大量结构：

- 语法规律
- 事实模式
- 因果表达
- 任务格式
- 人类写作习惯
- 问题和答案之间的对应关系

当模型在海量文本上反复学习“前文和后文的关系”时，它实际上也在学习：

- 什么叫定义
- 什么叫解释
- 什么叫论证
- 什么叫总结
- 什么样的文字通常跟什么问题匹配

所以你看到的“能力”，常常不是模型显式学会某个单独技能，而是大量语言模式叠加后的结果。

## 5. 输入、输出与概率分布

### 5.1 输入不是“意思”，而是 token 序列

人类读句子时会直接感受到意思，但模型接收到的其实是离散 token。

所以从模型角度看，输入更像：

- 一串符号
- 每个符号被映射成向量
- 模型在这些向量关系上做计算

### 5.2 输出也不是唯一答案，而是概率分布

模型每一步产生的不是一个固定词，而是一整组候选 token 及其概率。

例如在某一步，它可能认为：

- `好的` 概率 0.35
- `当然` 概率 0.22
- `可以` 概率 0.19

最后输出哪一个，还要看采样策略。

这就是为什么同一个问题有时会出现不同回答。

### 5.3 上下文会不断改变概率

只要前面任何一个 token 不同，后面的概率分布就可能变化。

所以 Prompt 工程并不是“神秘咒语”，本质上是在改变模型看到的条件上下文，从而改变后续概率分布。

## 6. 能力从哪里来

LLM 的能力通常来自下面几部分共同作用：

### 6.1 大规模数据

模型通过海量语料接触到丰富表达、知识模式和任务形式。

### 6.2 强大的序列建模结构

例如 Transformer，可以更有效建模远距离上下文关系。

### 6.3 大规模训练

参数优化让模型逐渐学会在不同上下文下给出更高概率的正确延续。

### 6.4 后训练

模型在预训练后通常还会经过指令微调、偏好对齐等阶段，变得更会“当助手”。

## 7. 模型为什么会犯错

理解失败原因，比只看成功案例更重要。

### 7.1 它追求的是高概率，不是真理

模型优化目标不是“保证事实正确”，而是“生成统计上更像正确答案的文本”。

### 7.2 上下文不完整

如果问题依赖外部资料，而上下文里没有，模型就可能猜。

### 7.3 训练分布和当前任务不一致

模型可能见过类似形式，但没见过你这个具体场景。

### 7.4 多步推理容易累积误差

一步错，后面会在错误前提上继续生成。

---

## 8. 初学者最容易混淆的点

### 8.1 “像理解”不等于“真的理解”

模型在很多场景下表现得像理解，但底层机制和人类理解并不相同。

### 8.2 “记住了知识”不等于“随时能稳定取出”

模型参数里可能包含相关知识，但在具体提问下不一定稳定激活。

### 8.3 “输出很流畅”不等于“逻辑很可靠”

语言流畅是它最擅长伪装的部分，也是最容易让人放松警惕的部分。

## 9. 练习题

### 9.1 思考题

为什么说“预测下一个 token”这个目标虽然简单，但足以支持问答、总结、翻译等多种能力？

参考思路：

因为这些任务都可以表示为“在给定上下文下生成合适后续文本”，而海量文本中已经隐含了大量任务模式。

### 9.2 判断题

“LLM 回答错误，说明它没有学过相关知识。”

参考答案：

不一定。也可能是问题表达方式、上下文不足、采样波动或多步推理误差导致没有把相关知识稳定调出来。

这篇文档最重要的一句话是：

**LLM 不是在数据库里查标准答案，而是在当前上下文条件下，按概率一步一步生成后续 token。**
