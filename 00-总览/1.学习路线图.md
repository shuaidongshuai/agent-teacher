这篇文档适合第一次进入本仓库，想先弄清整体路径再开始学习的人。

## 2. 学习目标

读完后，你应该能够：

1. 理解整套课为什么按当前顺序组织
2. 知道自己当前所在阶段
3. 知道每个阶段应该产出什么

## 3. 为什么这套课按这个顺序设计

这套课程默认采用下面的主线：

`LLM基础 -> Prompt工程 -> 工具调用 -> RAG -> Agent -> 评测与优化 -> 项目实战 -> 模拟面试`

原因是：

1. 先理解模型行为，再去谈任务设计
2. 先掌握工具接入，再理解知识供给
3. 先做好单能力链路，再进入多步决策系统
4. 先建立评测闭环，再让项目真正具备迭代价值

## 4. 四个学习阶段

### 第一阶段：打基础

目标：先建立对 LLM 的整体认知，不急着追框架。

建议掌握：

- Token、上下文窗口、采样、系统提示词
- Transformer 的基本直觉
- 预训练、指令微调、对齐的区别
- Prompt 的任务表达和结构化输出

阶段产出：

- 能解释“LLM 是怎么工作的”
- 能写出一份清晰 Prompt
- 能做一个结构化输出 Demo

### 第二阶段：从会问到会构建

目标：不只是和模型对话，而是开始做应用。

建议掌握：

- 函数调用与结构化参数设计
- 工具失败处理与结果回注
- RAG 基本链路
- 查询理解、召回、重排的分工

阶段产出：

- 一个工具调用 Demo
- 一个最小 RAG 系统
- 一份链路排障图

### 第三阶段：进入 Agent

目标：理解 Agent 不只是多轮对话，而是感知、决策、行动和状态推进。

建议掌握：

- 任务拆解与规划
- 工具选择与调度
- 记忆与状态管理
- Workflow、ReAct、Multi-Agent 的差异

阶段产出：

- 一个能处理多步骤任务的 Agent Demo
- 一份架构选型说明
- 一次失败复盘

### 第四阶段：评测与优化

目标：从“能跑”升级到“可靠”。

建议掌握：

- Prompt 评测
- RAG 评测
- Agent 轨迹分析
- 成本、速度、稳定性优化

阶段产出：

- 一套最小评测标准
- 一份版本对比记录
- 一份项目优化记录

## 5. 工程转型学习者的建议节奏

如果你是边工作边学，建议这样推进：

1. 每周只推进 1 个主主题
2. 每个主题至少配 1 个小练习或小 Demo
3. 每两周回头整理一次术语表
4. 每完成 1 个项目，就补 1 次失败复盘

## 6. 常见误区

- 一开始就跳到 Agent 或项目
- 只追热点，不补底层认知
- 只记成功案例，不记失败原因
