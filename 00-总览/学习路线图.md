# 学习路线图

## 第一阶段：打基础

目标：先建立对 LLM 的整体认知，不急着追框架。

建议掌握：

- Token、上下文窗口、采样、温度、系统提示词
- Transformer 的基本直觉
- 预训练、指令微调、对齐的区别
- 模型调用的基本方式

阶段产出：

- 能向别人解释“LLM 是怎么工作的”
- 能写出基础 Prompt
- 能调用一个模型 API 完成简单任务

## 第二阶段：从“会问”到“会构建”

目标：不只是和模型对话，而是开始做应用。

建议掌握：

- Prompt 模板设计
- 结构化输出
- 工具调用 / 函数调用
- RAG 基本链路

阶段产出：

- 一个小型问答助手
- 一个带工具调用的 Demo

## 第三阶段：进入 Agent

目标：理解 Agent 不只是“多轮对话”，而是“感知-决策-行动”。

建议掌握：

- 任务拆解与规划
- 工具选择
- 短期记忆与长期记忆
- ReAct、Workflow、Multi-Agent 的差异

阶段产出：

- 一个能处理多步骤任务的 Agent Demo
- 一份 Agent 失败案例复盘

## 第四阶段：评测与优化

目标：从“能跑”升级到“可靠”。

建议掌握：

- Prompt 评测
- RAG 评测
- Agent 轨迹分析
- 成本、速度、稳定性优化

阶段产出：

- 一套简单评测标准
- 一份项目优化记录
