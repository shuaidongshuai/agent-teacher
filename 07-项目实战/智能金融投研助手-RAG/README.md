## 1. 项目解决什么问题

这个项目面向金融 PDF 投研场景，目标是让你系统学习 RAG 在复杂文档场景里的核心链路与工程难点。

目标问题示例：

- 某公司去年的毛利率变化趋势及其原因？
- 过去三年研发投入变化与净利润趋势是否一致？
- 这份研报给出的核心投资逻辑是什么？

## 2. 为什么这个项目适合当前学习阶段

这个项目适合作为 **RAG 进阶项目**，因为它不是停留在“固定切片 + 向量检索”的最小 Demo，而是把注意力放在：

- PDF 解析
- 数据清洗
- 结构化切片
- 后续可扩展的检索、重排和评估

## 6. 运行方式

在仓库根目录执行：

```bash
python3 07-项目实战/智能金融投研助手-RAG/scripts/run_chunking_demo.py
```

这个 demo 会：

1. 加载一组模拟金融报告 block
2. 做清洗
3. 做结构化 + 语义切片
4. 把结果打印到终端

## 7. 推荐观察点

建议重点看：

1. 标题是不是强边界
2. 表格是不是单独成块
3. 相邻段落是否真的属于同一语义单元
4. chunk 上是否保留了页码、标题层级、块类型等 metadata

## 8. 常见失败原因

金融 RAG 最常见的失败点包括：

1. PDF 解析后的结构信号已经丢了
2. 清洗时把重要表格或标题信息一并删掉
3. 切片只按固定长度，忽略了语义边界
4. 检索阶段没有利用好年份、公司简称、指标名等特征
