## 1. 作业背景

本次作业配套项目为：

- `07-项目实战/agent-study-planner/main.py`

目标不是追求复杂功能，而是观察一个最小 Agent 是否真正体现了：

1. 目标理解
2. 任务规划
3. 执行建议
4. 结果反思

## 2. 作业 1 运行结果

### 输入 1：三天内入门 Python

观察结果：

- Agent 正确识别出时间限制为 3 天
- 计划被压缩为 3 个步骤
- 反思器指出缺少复盘环节

结论：

这个结果说明 Agent 已经能根据短时间窗口调整计划密度，但也暴露出一个特点：

**当时间非常短时，系统会倾向于先保核心任务，从而牺牲复盘步骤。**

### 输入 2：一周内完成 RAG 基础学习

观察结果：

- Agent 正确识别出时间限制为 7 天
- 计划输出为 4 个阶段
- 最后包含复盘与补缺

结论：

这说明当前规划器对“时间更充裕”的场景处理更稳定，能形成完整闭环。

### 输入 3：准备一次 20 分钟的 Agent 分享

观察结果：

- Agent 能识别产出物应该是讲稿或提纲
- 由于没有明确日期，系统默认按 7 天估算
- 计划更偏向“理解 + 拆解 + 产出 + 复盘”的通用模式

结论：

这说明当前系统已经能根据“分享”类任务调整输出物类型，但对更细的时间约束仍然比较粗糙。

## 3. 作业 2 规划质量观察

### 3.1 它有没有正确理解时间约束

有，基础情况是正确的。

例如：

- `三天内` 被识别为 3 天
- `一周内` 被识别为 7 天
- 没写时间时，默认估算为 7 天

但它目前还不能很好处理更复杂的表达，比如：

- 周末前完成
- 明晚前准备好
- 每天只能学 1 小时

### 3.2 它给出的产出物是否清晰

大多数情况下是清晰的，但清晰度取决于输入是否带有明确目标词。

例如：

- 输入中有 `Demo`，就会输出“最小可运行 Demo”
- 输入中有 `分享`，就会输出“讲稿或提纲”
- 如果只是“学习某主题”，产出物就会退化成较通用的“阶段化学习成果记录”

所以它已经有“产出导向”，但还不够精细。

### 3.3 它的步骤是否可执行

整体上可执行，但执行颗粒度还偏粗。

优点：

- 每一步都有目标
- 每一步都有动作
- 每一步都有产出

不足：

- 还没有细到“今天具体学哪篇资料、做哪道练习”
- 还没有结合每日可用时长来控制任务量

## 4. 作业 3 扩展能力建议

如果让我从题目里选一个扩展，我会优先选：

**增加保存结果到 Markdown 文件的功能。**

原因有 3 个：

1. 它能把 Agent 输出变成真正可交付产物
2. 它能帮助你理解“规划结果如何落地到文件系统”
3. 它是后续接日记、周报、学习档案的基础

如果做第二个扩展，我会选：

**增加每日学习时长输入。**

因为这样一来，Agent 的规划质量会明显提升。它不再只是按天粗分，而是能按真实可投入时间来安排任务密度。

## 5. 作业 4 复盘答案

### 5.1 这个 Agent 真正解决了什么问题

它解决的是：

**把一个模糊的学习目标，初步转成一份能执行、能检查的计划。**

很多时候，学习失败不是因为资料少，而是不知道先做什么、做到什么程度算完成。

这个 Agent 的价值就在这里。

### 5.2 它最像 Agent 的地方是什么

它最像 Agent 的地方在于，它不是一次性给出建议，而是经过了：

1. 目标分析
2. 任务规划
3. 执行建议补充
4. 自检反思

这已经具备了一个最小 Agent 的闭环结构。

### 5.3 它最不像 Agent 的地方是什么

它最不像完整 Agent 的地方在于：

- 没有真实外部工具交互
- 没有长期记忆
- 没有根据执行结果再次调整计划
- 反思还只是静态检查，不是真正的动态反馈

也就是说，它更像“教学型 Agent 骨架”，而不是“生产级执行 Agent”。

### 5.4 如果继续升级，我下一步最想加什么

我最想加的是：

**保存结果到 Markdown 文件，并支持多轮修改。**

因为这是从“看输出”走向“交付结果”的关键一步。

一旦这一步完成，后面再接：

- 历史记录
- 计划对比
- 周期复盘
- 工具调用

都会更自然。

## 6. 总结与下一步建议

这份作业答案最重要的结论不是“项目能跑”，而是：

**这个项目已经足够用来理解 Agent 的最小闭环，但还远没有达到真实复杂任务的工程强度。**

下一步建议你这样继续：

1. 自己先改一个小功能
2. 再增加一个真实工具调用
3. 最后尝试让 Agent 根据执行结果二次修正计划

一句话总结：

**会运行只是开始，会观察、会复盘、会升级，才是真正的 Agent 实战。**
