## 1. 本章定位

`07-项目实战` 解决的是：

**怎么把前面的知识真正变成可运行、可观察、可复盘、可讲述的项目。**

这一章不是让你把所有项目都跑一遍，而是帮你选对项目、按难度递进，并形成能对外授课和对内复盘的项目材料。

## 2. 适用人群

适合：

- 已看过前面理论内容，准备开始做作品的人
- 想通过项目把 RAG、工具调用、Agent 串成完整能力链的人

## 3. 学习目标

学完这一章后，你应该能够：

1. 根据自己的阶段选择合适难度的项目
2. 理解不同项目分别训练哪些能力
3. 在项目中记录运行、排错、评测和复盘过程
4. 形成可讲述、可面试表达的项目经历

## 5. 推荐阅读与实战顺序

建议按下面顺序推进：

1. 先读 [1.项目练习清单.md](/Users/chenmingdong01/Documents/AI/agent/07-项目实战/1.项目练习清单.md)
2. 先做一个 L1 的 MCP 或工具项目
3. 再做一个 L2 的 RAG 项目
4. 再进入 L3 的单 Agent 项目
5. 最后再挑战 L4 的规划执行分离或多 Agent 项目

## 6. 前置知识要求

不同等级项目对应的前置知识如下：

- L1：至少完成 `02-Prompt工程` 和 `04-工具调用与函数调用`
- L2：至少完成 `03-RAG`
- L3：至少完成 `05-Agent`
- L4：建议同时完成 `05-Agent` 和 `06-评测与优化`

## 7. 学完后的产出

建议每个项目至少留下下面 4 类产出：

1. 项目目标与架构说明
2. 运行与排错记录
3. 失败案例与优化思路
4. 可对外讲述的项目复盘稿

## 8. 常见误区

- 一开始就冲最复杂的多 Agent 项目
- 只跑成功路径，不记录失败链路
- 只看代码，不做项目讲述和复盘
- 做完项目后不回到评测章节补闭环

## 9. 总结与下一步

项目实战的价值，不是让你“跑过几个仓库”，而是让你真正理解系统为什么能跑、为什么会失败、哪里值得改。
