## 1. 适用人群

这部分内容适合已经学完 LLM、Prompt、RAG、工具调用和 Agent 基础，希望从“会搭建”进入“会判断、会定位、会优化”的学习者，尤其适合：

- 已经做过 Demo，但不知道系统到底是好是坏的人
- 经常靠手测和感觉改 Prompt、改模型、改流程的人
- 想建立 AI 项目评测体系和优化闭环的人

## 2. 学习目标

学完这一章后，你应该能够：

1. 理解评测为什么是 AI 工程中的关键分水岭
2. 为 Prompt、RAG、Agent 构建最小可用评测集
3. 把问题拆成任务理解、检索、工具、状态、生成等不同层
4. 基于失败样本做更有针对性的优化，而不是盲调
5. 建立“评测 -> 定位 -> 改进 -> 回归验证”的闭环意识

## 3. 这一章在整个学习路线里的位置

很多人学到 `05-Agent` 或做完 `07-项目实战` 后，会遇到一个共同问题：

系统明明“能跑”，但你不知道：

- 它到底稳定不稳定
- 为什么有时好、有时差
- 哪个优化动作真正有效
- 新版本是不是把旧问题又带回来了

这时就必须进入 `06-评测与优化`。

如果说：

- `01-05` 主要解决“怎么理解和搭建系统”
- `07` 主要解决“怎么做项目和跑 Demo”

那么 `06` 解决的就是：

**如何把 AI 系统从主观试用，推进到可评估、可定位、可持续优化。**

## 6. 这一章最重要的认知主线

如果你只记住这一章的一条主线，可以记住下面这句话：

**AI 系统的优化不是从“开始改”开始，而是从“先判断问题在哪一层”开始。**

可以进一步拆成下面 4 个判断：

### 6.1 没有评测，就没有稳定优化

只靠主观试用，往往无法判断系统是真的变好，还是只是刚好撞上了顺手样本。

### 6.2 不同系统要分层评测

Prompt、RAG、Agent 解决的是不同问题，不能只共用一套“最终答案对不对”的判断方式。

### 6.3 失败样本比成功样本更有价值

真正推动系统进步的，通常不是漂亮案例，而是那些反复暴露短板的失败样本。

### 6.4 优化必须回到固定样本集验证

否则你很难区分：

- 是本次改动有效
- 还是测试问题碰巧更简单

## 7. 学习建议与下一步

学习这一章时，建议你不要只当成概念阅读，而是直接拿你自己的一个项目来对照：

1. 先准备一批最小评测样本
2. 为每条样本定义成功标准
3. 跑一次当前版本
4. 挑 3 个失败样本做链路复盘
5. 做一次最小改动后重新验证
