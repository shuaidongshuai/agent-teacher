## 面试题与参考答案

### 1. 一个好的 Prompt，通常应该包含哪些部分？

**参考答案：**

一个好 Prompt 不一定很长，但通常会把任务边界说清楚。常见组成包括：

- 角色或身份
- 任务目标
- 输入背景
- 约束条件
- 输出格式
- 示例

### 2. Few-shot 示例为什么有用？

**参考答案：**

Few-shot 的作用是“用例子告诉模型你想要什么”。很多时候，模型不是不懂任务，而是不确定你想要的风格、结构和判断标准。示例能帮助模型快速对齐输出方式。

它尤其适合：

- 分类标准不够显式的任务
- 输出格式要求严格的任务
- 你希望模型模仿某种写法的任务

### 3. 为什么明明写了 Prompt，模型结果还是不稳定？

**参考答案：**

常见原因有：

- 任务本身定义不清
- 输入信息不完整
- 输出标准不明确
- Prompt 与任务复杂度不匹配
- 温度等参数设置不合适
- 工具结果或外部上下文本身有噪声

### 4. 如果让你调一个效果不好的 Prompt，你会怎么做？

**参考答案：**

可以按一个很实用的顺序回答：

1. 先明确失败现象，别笼统说“效果不好”
2. 收集失败样本，判断是理解错、格式错还是事实错
3. 缩小变量，一次只改一项
4. 补充约束、示例或输出模板
5. 建立小规模测试集做对比验证
